DEV NOTES
Rules developers must preserve
Avatar rule (locked):
All user avatars on the Island and in themed rooms are round tiles with feet. The feet are a defining visual element of the Lovehuddle aesthetic — do not remove them, do not replace with shadows, do not flatten to plain circles.
Worldwide launch (locked):
Region eligibility, ID document acceptance, distance units, and currency must all be locale-aware. The blueprint's UK-only language is Phase-1-only and is overridden by this wireframe.
⚠️ Unresolved contradiction (gift mechanic):
Two rule sets exist for gifts and voice inventory. Screens marked [!] below implement the detailed version (counters, inventory, "use now / save for later", pack-required pop-ups). The earlier client rule said "no numbers, no caps, no counters, no paywalls visible to user." Both are shown so the contradiction is visible at handover. Confirm which version to ship before build.
⚠️ New feature (not in blueprint) — Mute User:
The blueprint defines only Block as the audio-silencing mechanism (permanent, identity-locked). This wireframe adds Mute as a softer, session-only alternative (silences a user's audio locally for the current Huddle only). Mute screens are marked [!]. Confirm before build, or remove if Block-only is preferred.
📖 Vocabulary aliases (one feature, two names):
Some screens use two names for the same feature to satisfy review feedback. Do not build duplicate features.
  • Favourites = Saved Users (one private list, blueprint §9.2). Tapping "★ Save user" in the User Menu adds to this list.
  • Invite to Huddle = Room Icon flow (blueprint §1, screen 16b). When User A taps "Invite to Huddle" on User B's profile, B receives a Room Icon notification.
  • Connect means the post-Huddle private decision (blueprint §9, screen 22). It is not a profile button. To start a chat with someone, the flow is: Save → Request → Accept → Chat (blueprint §9.3–9.4, screens 16, 24a, 25, 26).
🏷 Huddle naming convention:
Wherever a Huddle is referenced in the UI, use the dynamic pattern {RoomName} Huddle — populated from the chosen room's name (Coffee, Music, Pets, etc., per the Rooms list, screen 14).
  • Incoming invite (screen 00g): "Kevin invited you to the {RoomName} Huddle"
  • Voice Huddle header (screen 19): "{RoomName} Huddle" shown above the timer
  • Rooms list (screen 14): each room becomes a {Name} Huddle when entered
Do not hardcode any single room name as the canonical Huddle label.
