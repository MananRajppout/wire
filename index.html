SYSTEM NOTES
Backend logic · developer annotations · no UI screens
SYSTEM ANNOTATION · SYS-A
Huddle spawning
Rule
If a Huddle reaches 20 users, the system automatically spawns a new Huddle instance for the next arrivals.
User experience
Users never see "Huddle 1", "Huddle 2" etc. They simply enter a calm room. Spawning is invisible.
No UI change needed. Backend only.
SYS-AHuddle spawning
SYSTEM ANNOTATION · SYS-B
Matching & routing
Users routed into Huddles based on:
1. Sexual preference
2. Distance range
3. Shared favourite rooms (≥1)
4. Age dealbreakers
5. Room interest bonus (optional)
Fallback
If no exact match → broaden distance. Never match across incompatible preferences.
No algorithmic ranking. No behaviour weighting.
SYS-BMatching logic
SYSTEM ANNOTATION · SYS-C
AI welcome line
When Huddle activates
Show a calm, room-themed AI icebreaker at the start of the Voice phase. Shown once per Huddle. Never repeats for the same user.
Examples by room
☕ Coffee: "What's your perfect morning routine?"
⚽ Sports: "Biggest match you ever watched live?"
🐾 Pets: "What's the funniest thing your pet has done?"
Calm · human · non-gamey. Blueprint §5.
SYS-CAI welcome line
SYSTEM ANNOTATION · SYS-D

Runs continuously across ALL surfaces
All 14 rooms · Main Huddle · any future huddles · any future group spaces

Safety-only · GDPR-compliant · purpose-limited. No UI needed.
